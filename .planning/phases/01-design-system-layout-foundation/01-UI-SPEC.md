| `--secondary` | `#F1F5F9` | Secondary surface (slate-100) |

### 60/30/10 Application

| Split | Color | Surfaces |
|-------|-------|----------|
| 60% dominant | `#FAFAFA` (background) | Page background, main content area |
| 30% secondary | `#FFFFFF` / `#F1F5F9` (card / muted) | Section cards, TrustBar strip, alternating section backgrounds, navigation |
| 10% accent | `#FB5B3D` (primary orange) | See reserved list below |

### Accent Reserved For (exhaustive list)

- Primary CTA buttons (solid fill)
- Navigation CTA button ("Book a Call")
- Section heading decorative accent marks (left border or underline)
- Hover state on interactive text links
- Focus ring on form inputs
- Active/current state indicators in navigation

Accent is NOT used on: body text, section backgrounds, card borders, icons in general, decorative elements.

Deferred to Phase 9 per CONTEXT.md. Dark mode CSS variables already exist in globals.css and will be refined then. No dark mode work in this phase.

## Animation Contract

Source: CONTEXT.md decisions + existing `lib/animations.ts`

### Retained Variants

| Variant | Duration | Easing | Trigger | Usage |
|---------|----------|--------|---------|-------|
| `fadeIn` | 300ms | easeOut | Intersection Observer (once) | Default section entrance |
| `fadeInUp` | 400ms | easeOut | Intersection Observer (once) | Content blocks within sections |
| `staggerContainer` | stagger 50ms, delay 100ms | -- | Parent wrapper | Groups of cards, list items |
| `staggerItem` | 300ms | easeOut | Child of stagger | Individual card/item entrance |
| `buttonHover` | 200ms | easeInOut | hover/tap | CTA buttons: scale 1.05 on hover, 0.95 on tap |
| `drawerAnimation` | 300ms | easeInOut | open/close | Mobile navigation drawer |

### Removed Variants

| Variant | Reason |
|---------|--------|
| `floatingOrb` | Conflicts with clean/minimal directive (CONTEXT.md) |
| `scaleOnHover` | Redundant with `cardHover`; cards should not scale |
| `cardHover` | Cards use subtle border/shadow transition instead of scale |
| `counterAnimation` | No counter UI in Phase 1 |
| `slideInFromLeft` | Unnecessary directional animation; fadeInUp is sufficient |
| `slideInFromRight` | Same as above |
| `letterReveal` | Over-designed; not content-enhancing |
| `textReveal` | Redundant with fadeInUp |
| `progressLine` | No SVG progress line in redesigned Process section |

### New Variant

| Variant | Duration | Easing | Usage |
|---------|----------|--------|-------|
| `sectionFadeIn` | 500ms | `[0.25, 0.1, 0.25, 1]` (ease) | Top-level section wrapper; Intersection Observer with `threshold: 0.1`, `triggerOnce: true` |

### Animation Rules

1. Every animation triggers exactly once (no repeat, no reverse)
2. No animation on elements above the fold (hero text appears immediately; stagger for sub-elements only)
3. No decorative/ambient animation (no floating, pulsing, or continuous motion)
4. `prefers-reduced-motion: reduce` disables all motion -- elements appear in final state

## Section Architecture

Order per CONTEXT.md decision (buyer decision journey):

| Order | Component | New/Rewrite/Keep | Purpose |
|-------|-----------|-------------------|---------|
| 1 | `Navigation` | Keep (update) | Top nav with logo, links, CTA button |
| 2 | `Hero` | Rewrite | Value proposition, primary CTA |
| 3 | `TrustBar` | New | Lightweight credibility strip (stats or trust markers) |
| 4 | `ProblemSection` | New | Target audience pain points (replaces Introduction) |
| 5 | `Services` | Rewrite | Service offerings overview |
| 6 | `Process` | Rewrite | Engagement process steps |
| 7 | `ResultsSection` | New | Before/after outcomes (placeholder) |
| 8 | `About` | Rewrite | Personal story, practitioner credibility |
| 9 | `CTA` | Keep (update) | Final conversion call-to-action |
| 10 | `Footer` | Keep (update) | Links, copyright |

### Layout Rules

- Max content width: 1200px (centered)
- Section padding: 64px vertical (desktop), 32px vertical (mobile)
- Horizontal page padding: 24px (mobile), 48px (tablet), 0 (desktop, content width handles it)
- Alternating section backgrounds: odd sections use `--background`, even sections use `--muted` to create visual rhythm
- No sidebar layout -- single column, full-width sections
