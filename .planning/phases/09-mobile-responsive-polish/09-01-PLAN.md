   - In Mobile Drawer (the `fixed top-0 right-0` div), add `<ThemeToggle />` at the top of the drawer content (before nav items), styled inline within a flex container so it appears at the top-right of the drawer

2. Fix `components/Footer.tsx` dark mode:
   The footer uses `bg-foreground text-background` which creates an inverted section. In dark mode, `foreground` becomes light (#F8FAFC) and `background` becomes dark (#0F172A), causing the footer to appear light-on-dark-text — the exact opposite of intended.

   Fix: Replace the footer's color approach with explicit dark mode handling:
   - Change outer `<footer>` from `bg-foreground text-background` to `bg-slate-900 text-slate-100 dark:bg-slate-950 dark:text-slate-200`
   - Change `text-background/70` references to `text-slate-400`
   - Change `text-background/60` references to `text-slate-500`
   - Change `text-background` references (for headings/hovers) to `text-white`
   - Change `border-background/20` to `border-slate-700`

   This ensures the footer always looks like a dark section in both themes, which is the standard pattern for footers.

3. After changes, verify no other components have similar inverted color patterns by searching for `bg-foreground` usage. All other components use standard `bg-background`/`bg-muted`/`bg-card` which will work correctly with the CSS variable swap.
