# Requirements: Infinite Play

**Defined:** 2026-04-05
**Core Value:** A site that makes it immediately obvious what Jeremy does (Claude implementation consulting), feels authentic and direct, and generates inbound leads.

## v1 Requirements

Requirements for initial release. Each maps to roadmap phases.

### Positioning & Copy

- [ ] **COPY-01**: Clear value proposition above the fold — visitor understands who Jeremy helps and what he does within 5 seconds
- [ ] **COPY-02**: Target audience clarity — copy speaks to SMB founders, mid-market teams, and knowledge workers so each can self-identify
- [ ] **COPY-03**: Direct, no-hype tone throughout — no corporate jargon, no urgency tactics, no AI buzzwords
- [ ] **COPY-04**: Claude product expertise taxonomy — explicit coverage of Chat, Teams, Enterprise, Claude Code, API, MCP showing depth generalists can't match
- [ ] **COPY-05**: Practitioner credibility — real examples, before/after workflows, proof of daily Claude usage (not just consulting about it)

### Services & Business Model

- [ ] **SERV-01**: Services section with 3-4 clear offerings — outcome-focused naming, not capability-focused
- [ ] **SERV-02**: "How to work with me" engagement process — 3-4 step visual (Talk > Plan > Execute > Results)
- [ ] **SERV-03**: Contact form and email — multiple paths to reach Jeremy (form, email, booking)
- [ ] **SERV-04**: Packaged service tiers with "starting from" pricing — Quick-Start ($1.5-3K), Implementation Sprint ($5-15K), Retainer ($2-5K/mo)
- [ ] **SERV-05**: Audience-specific entry points — distinct messaging for each target audience so they see themselves

### Conversion & Lead Gen

- [ ] **CONV-01**: Primary CTA: Book a discovery call via Calendly integration — appears in hero and after each major section
- [ ] **CONV-02**: Mobile-responsive design — all content works on mobile devices
- [ ] **CONV-03**: SEO metadata and social cards — updated for new Claude implementation consulting positioning
- [ ] **CONV-04**: Email capture with value exchange — lead magnet (Claude setup checklist or similar) gated behind email
- [ ] **CONV-05**: Availability indicator — "Currently accepting new clients" or "Next available: [month]" creating honest urgency

### Trust & Credibility

- [ ] **TRUST-01**: Professional headshot and personal bio — authentic photo, short bio proving practitioner credibility
- [ ] **TRUST-02**: Social proof / credibility markers — testimonials, results numbers, or "helped X teams adopt Claude" even if starting small
- [ ] **TRUST-03**: Inline case studies / results — 2-3 short before/after stories (stubs initially, filled as engagements complete)
- [ ] **TRUST-04**: Content demonstrating expertise — static practical content showing real Claude knowledge (not a blog system)

### Design & Technical

- [ ] **DSGN-01**: Clean, minimal website redesign — content-first, no clutter, personal brand aesthetic
- [ ] **DSGN-02**: Updated analytics with Plausible — privacy-friendly, no cookie banners, actionable dashboard
- [ ] **DSGN-03**: Dark mode support — common user expectation in 2026
- [ ] **DSGN-04**: Section architecture following buyer decision journey — Hero > Trust > Problem > Services > Process > Results > About > CTA
- [ ] **DSGN-05**: Form handling with react-hook-form + zod — client and server validation with same schema
- [ ] **DSGN-06**: Email notifications via Resend — confirmation and notification emails for form submissions
- [ ] **DSGN-07**: Framer Motion animations refined — purposeful animations that enhance content, not distract

## Out of Scope

Explicitly excluded. Documented to prevent scope creep.

| Feature | Reason |
|---------|--------|
| AI chatbot on the site | Gimmicky; bad chatbots hurt credibility; visitors want to talk to Jeremy |
| Client portal / dashboard | Way too much scope for a marketing site; use Notion/Google Docs for client work |
| Pricing calculator | Custom consulting doesn't calculate well; creates anchoring problems |
| Case study database | Requires completed engagements to populate; empty databases look worse than none |
| Multi-page service microsites | Fragments user journey; solo consultant sites work best focused |
| Exit-intent popups | Annoying, feels desperate, contradicts authentic brand |
| Urgency/scarcity tactics | "Launch price!" "Only 3 spots left!" — this is the shady AI influencer vibe Jeremy rejects |
| Fake certification badges | If not a certified Anthropic partner, don't imply it; lead with real practitioner credibility |
| OAuth / user accounts | This is a marketing site, not a web app |
